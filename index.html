Manual do Git

Bem-vindo ao "Manual do Git"! Este guia abrangente foi criado para ajudar iniciantes a aprender e dominar o Git, um sistema de controle de versão essencial para o desenvolvimento de software. Se você está iniciando sua jornada no controle de versão ou deseja aprimorar suas habilidades no Git, você está no lugar certo.

O que é o Git?

O Git é um sistema de controle de versão distribuído, o que significa que ele permite que várias pessoas trabalhem em um projeto de software ao mesmo tempo. Ele rastreia todas as alterações feitas nos arquivos do projeto e armazena um histórico completo de cada modificação. Isso facilita a colaboração, o rastreamento de problemas e a recuperação de versões anteriores do código.

Importância do Git no Desenvolvimento de Software

  • colaboração Eficiente: O Git permite que vários desenvolvedores trabalhem juntos em um projeto, combinando suas alterações de forma organizada.
  • Histórico Detalhado: Cada alteração é registrada, permitindo que você veja quem fez o que e quando. Isso é valioso para solucionar problemas e entender o desenvolvimento do projeto.
  • Controle de Versão: O Git oferece controle de versão, o que significa que você pode facilmente retornar a versões anteriores do código, caso algo dê errado.
  • Ramificações e Mesclagens: O Git permite criar ramificações (branches) para trabalhar em novos recursos ou correções de bugs sem afetar o código principal. Posteriormente, essas ramificações podem ser mescladas (merged) de volta ao código principal.
  • Armazenamento em Nuvem: O Git é frequentemente usado com plataformas de hospedagem na nuvem, como GitHub, GitLab e Bitbucket, tornando o código acessível e seguro.

Introdução à Configuração Inicial

Configurar seu nome de usuário e endereço de email no Git é o primeiro passo para começar a usar esta poderosa ferramenta de controle de versão. É importante, pois isso permite que o Git associe suas contribuições aos seus detalhes pessoais, tornando-o um colaborador identificável em projetos de código aberto e em equipes de desenvolvimento.

Configurando Nome de Usuário

Vamos começar a configurar seu nome de usuário no Git. Isso garantirá que suas contribuições aos repositórios sejam atribuídas a você. Para definir seu nome de usuário, use o seguinte comando:

git config user.name "Seu Nome"

Configurando endereço de e-mail

O próximo passo é configurar seu endereço de email, que também é essencial para identificar você como o autor das alterações. Use o comando a seguir para definir seu endereço de email:

git config user.email "seu@email.com

Verificação de Configuração

Após configurar seu nome de usuário e endereço de e-mail, é importante verificar se as configurações foram aplicadas corretamente. Use o comando abaixo para listar suas configurações atuais:

git config --list

Trabalhando com repositórios

Repositórios são fundamentais no Git. Eles são onde o código-fonte e as alterações são armazenados. Aprenda a criar repositórios do zero e a clonar repositórios existentes.

Criando Repositório

  1. Inicialize um Novo Repositório: Use git initpara criar um novo repositório Git.
  2. Adicione Arquivos: Use git add para incluir os arquivos no controle de versão.
  3. Faça um Commit: Grave um commit inicial com uma mensagem descritiva com git commit -m "mensagem descritiva"

Clonando um Repositório

Clonando um repositório existente: Use git clone [URL] para criar uma cópia local de um projeto

Gerenciamento de Repositórios Remotos:

Adicionar um repositório remoto (origin): Use git remote add origin [URL]

Renomear um repositório remoto: Use git remote rename origin novo-nome

Remover um Repositório Remoto: Use git remote rm origin

Gerar um resumo de commits: Use git shortlog

Gerenciamento de Arquivos

O Gerenciamento de Arquivos é uma parte fundamental do uso do Git. Envolve a adição, confirmação e rastreamento de arquivos em um repositório Git. Compreender esse processo é crucial para o desenvolvimento colaborativo de software e a criação de um histórico de alterações consistente.

Tópico Descrição Comandos Git
Adicionar Arquivos Adiciona arquivos ao repositório git add 'nome arquivo'
Fazer um Commit Confirma as alterações em um commit git commit -m "Mensagem descritiva"
Histórico de Commits Mantem um registro de todos os commits git log

BRANCHES

git branch

Liste todos os ramos (branches) disponíveis no repositório.

git checkout [branch]

Mude para um ramo específico.

git merge [branch]

Mesclar alterações de um ramo em outro.

git push origin [branch]

Envie um ramo local para o repositório remoto.

Resolução de Conflitos

Conflitos podem ocorrer ao mesclar alterações de ramos diferentes. Neste tópico, você aprenderá a identificar, resolver e evitar conflitos de mesclagem, garantindo um fluxo de trabalho suave e colaboração eficiente.

Resolvendo conflitos:

  1. Após a detecção de conflitos, você precisará abrir os arquivos em um editor de código e resolver manualmente as áreas conflituosas. As áreas em conflito serão claramente marcadas com marcadores especiais, indicando as alterações do seu ramo e as alterações do outro ramo.
  2. Você deve editar o código para que ele faça sentido após a mesclagem. Isso envolve escolher quais alterações manter, quais descartar e como as alterações devem ser organizadas.
  3. Após resolver os conflitos, você deve adicionar os arquivos alterados ao índice com git add.

Comando do git

git merge [branch] Inicia a resolução de conflitos quando ocorrerem conflitos de mesclagem.

Personalização e Boas Práticas (Ignorando Arquivos)

Personalizar o Git para atender às suas necessidades e aplicar boas práticas é crucial. Além disso, aprender a ignorar arquivos indesejados é importante para manter seu repositório limpo. Neste tópico, você encontrará dicas sobre personalização e criação de arquivos .gitignore

.gitignore

Cria um arquivo .gitignore para especificar quais arquivos ou diretórios devem ser ignorados pelo Git.

git commit -am "Mensagem"

Adiciona automaticamente todos os arquivos modificados e faz um commit com uma mensagem

git reset HEAD~1

Desfaz o último commit, mantendo as alterações nos arquivos na área de preparação (staging).

git reset HEAD~1 --soft

Desfaz o último commit e mantem as alterações nos arquivos no diretório de trabalho.
